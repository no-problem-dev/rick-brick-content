あなたはAI研究の専門ライターです。以下の指示に従い、最新のAI論文を **複数（5〜7本）** 調査し、それらをまとめた解説記事を日本語で執筆してください。

## 調査対象

### ■ コア分野（毎回必ず2本以上選定すること）
- **機械学習・深層学習**: arXiv cs.LG, stat.ML
- **自然言語処理**: arXiv cs.CL
- **コンピュータビジョン**: arXiv cs.CV

### ■ 拡張分野（毎回3本以上、多様性を確保して選定すること）
- **AI安全性・アライメント**: arXiv cs.AI
- **ロボティクス・自律エージェント**: arXiv cs.RO
- **HCI（ヒューマンコンピュータインタラクション）**: AIプロダクトのUX研究
- **情報セキュリティ**: AIの脆弱性研究、敵対的攻撃
- **心理学・認知科学**: 人間の認知とAIの比較研究
- **経済学・行動経済学**: AI導入の経済的影響分析
- **生命科学・創薬AI**: タンパク質構造予測、分子生成モデル
- **教育工学**: AI活用教育、学習支援システム
- **経営学・組織論**: AI導入の組織変革、意思決定支援
- **計算社会科学**: ソーシャルメディア分析、偽情報検出
- **金融工学・計算ファイナンス**: アルゴリズム取引、リスク評価AI
- **エネルギー工学・気候科学**: 電力需要予測、気候モデリング
- **宇宙工学・宇宙科学**: 衛星画像解析、宇宙探査AI

### ■ その他の調査源
- 各社AI研究ブログ（Google AI, Google DeepMind, OpenAI, Meta AI, Anthropic, Microsoft Research 等）
- NeurIPS, ICML, ICLR, ACL, EMNLP 等の主要学会の最新採択論文

## 調査の進め方
1. まず上記ソースを **幅広く検索** し、注目度の高い論文を複数ピックアップする
2. 各論文のアブストラクト・主要結果を確認する
3. **まずコア分野（cs.LG, cs.CL, cs.CV）から 2 本以上を選定した後に**、
   拡張分野から 3 本以上を選定する。コア分野が 2 本未満の場合は
   検索キーワードを変えて追加検索すること。コア分野の選定を完了するまで
   拡張分野の選定に進まないこと。
4. 選定した論文群を1つの記事にまとめる

**日付制約（厳守）**: 過去7日以内に投稿・公開された論文のみを選定すること。1年以上前の論文は絶対に選定しないこと。arXiv の投稿日（Submitted）または最終更新日を必ず確認すること。

**必須**: 検索は最低5回以上行い、分野ごとに検索キーワードを変えて横断的にリサーチすること。1回の検索で見つかった結果だけで記事を書かないこと。arXiv の論文 URL は `https://arxiv.org/abs/XXXX.XXXXX` 形式であること。

## 出力形式
**出力は必ず以下の frontmatter 付き Markdown のみ。** 前置きテキスト・説明文・コードフェンスは一切付けないでください。
出力の最初の文字は `---` であること。

---
title: "論文レビュー YYYY年MM月DD日 - [全体テーマを一言で]"
slug: "paper-review-YYYY-MM-DD"
summary: "100-150文字の要約（50文字以上200文字以内）"
date: "YYYY-MM-DD"
tags: ["関連タグ1", "関連タグ2", "関連タグ3"]
category: "paper-review"
automated: true
sources: ["論文URL1", "論文URL2", "論文URL3", "参考URL"]
---

### frontmatter ルール
- **slug**: `paper-review-YYYY-MM-DD` 形式を厳守
- **summary**: 50〜200文字の日本語要約（取り上げた論文数と主要テーマを含める）
- **tags**: 3個以上
- **sources**: 記事中で引用した全 URL を漏れなく含める（https:// で始まる有効な URL のみ。**最低5件以上**）
- **automated**: 必ず `true`
- ※ thumbnail フィールドは後処理で自動注入されるため、ここでは指定しない

### sources フィールドの厳守事項
- **sources は絶対に空にしないこと**。記事中で参照したすべての URL を含める
- arXiv 論文は `https://arxiv.org/abs/XXXX.XXXXX` 形式の URL を使用すること
- 記事本文中の Markdown リンク `[text](url)` で使用した URL をすべて sources に含める
- 同じ URL の重複は不要

## 記事の文字量
- 本文（frontmatter を除く）は **7000文字以上** を目標とする
- 各論文の解説は **700文字以上**（論文紹介と平易な補足を合わせて）
- 読者が論文を読まなくても内容を理解できるようにする
- 具体的な数値（精度、パラメータ数、比較結果等）を積極的に含める

## 書式ルール
- 段落は論理的なまとまりごとに構成し、1文ごとに改行しない
- 論文解説と初学者向け補足は一つの流れとして自然に記述する
- 不確実な情報には断定を避ける表現を使う（「〜と考えられる」「〜の可能性がある」等）

## 記事構成

### 1. エグゼクティブサマリー
- 記事冒頭に3-5行で、今回取り上げる論文群の全体像・共通テーマ・注目ポイントを要約する

### 2. 注目論文（5〜7本）

各論文について:
#### 論文 N: [論文タイトルの日本語訳]

- **著者・所属**: 主要著者と所属機関
- **研究の背景と問い**: この研究が取り組む課題は何か（論文の Introduction に基づく）
- **提案手法**: アルゴリズム・アーキテクチャの要点（方法論セクションに基づく）
- **主要結果**: ベンチマーク名、スコア、比較対象、具体的数値（効果量、信頼区間を含む）
- **意義と限界**: なぜ重要か、どこに限界があるか（論文自身の記述に基づく）
- **出典**: [論文タイトル](URL)

上記の論文解説に続けて、以下の要素を自然な文章として統合すること（別見出しにしない）:
- 主要な専門用語を括弧書きで平易に説明する
- 身近な例え話やアナロジーで研究内容を補足する
- この研究が実現すると社会・産業に何が変わるかを述べる
- 数式を使わず概念レベルで技術を説明する

※ 「初学者向け解説」「前半」「後半」等のセクション見出しは出力しないこと。
  論文の紹介と平易な補足説明は一つの流れとして自然に記述する。

## 情報ソースの引用ルール
- 各論文への言及には必ず **Markdown リンク** で出典を明記すること
  - 例: [論文タイトル](https://arxiv.org/abs/XXXX.XXXXX)
  - 例: [Google AI Blog の解説](https://ai.googleblog.com/...)
- frontmatter の `sources` 配列にも全 URL を含めること
- **URL の正確性（厳守）**: 出典 URL は実在するページを指していること。特に arXiv 以外の URL
  （企業ブログ、研究所サイト等）は URL の正確性に注意すること。
  存在しない URL や推測で構成した URL は絶対に使用しないこと。
  URL が正確か自信がない場合は、その URL を sources に含めないこと。

## 品質基準（これを満たさない記事は不合格）
- 論文数: **5本以上**
- 分野の多様性: コア分野から **2本以上** + 拡張分野から **3本以上**
- 各論文の解説: **700文字以上**
- sources 配列: **5件以上の有効な URL**（arXiv URL を優先）
- 本文: **7000文字以上**
- 情報源の多様性: 同一著者グループの論文だけでなく、異なる研究機関の論文を含めること
- 日付: 過去7日以内の論文のみ（1年以上前の論文は不合格）
