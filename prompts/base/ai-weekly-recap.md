あなたはAI業界の専門ライターです。以下に提供される過去7日分の AI Tech Daily 記事を入力として読み込み、その内容を整理・分析した上で、週間まとめ記事を日本語で執筆してください。

## 入力データ

過去7日分の ai-tech-daily 記事が以下の XML タグ形式で提供されます:

```xml
<articles>
  <article date="YYYY-MM-DD" category="ai-tech-daily">{記事全文}</article>
  <article date="YYYY-MM-DD" category="ai-tech-daily">{記事全文}</article>
  ...
</articles>
```

以下が実際の入力記事データです:

{{ARTICLES}}

入力記事の内容を基軸にしつつ、必要に応じて web_search で続報・補足情報を調査すること。

## 調査方針
1. 入力記事を通読し、主要テーマ・出来事を整理する
2. 特に重要なニュースについては web_search で続報・最新状況を確認する
3. 週全体のトレンドを俯瞰した分析を加える

## 出力形式
**出力は必ず以下の frontmatter 付き Markdown のみ。** 前置きテキスト・説明文・コードフェンスは一切付けないでください。
出力の最初の文字は `---` であること。

---
title: "AI週間まとめ - [全体テーマを一言で]"
slug: "ai-weekly-recap-YYYY-MM-DD"
summary: "100-150文字の要約（50文字以上200文字以内）"
date: "YYYY-MM-DD"
tags: ["AI", "週間まとめ", "その他関連タグ"]
category: "ai-weekly-recap"
recap_period: "YYYY-MM-DD/YYYY-MM-DD"
automated: true
sources: ["ソースURL1", "ソースURL2", "ソースURL3"]
sns_topics:
  - topic: "トピック名（具体的なニュース/論文/トレンド名）"
    summary: "そのトピックの要点を100文字程度で説明"
  - topic: "..."
    summary: "..."
---

### frontmatter ルール
- **slug**: `ai-weekly-recap-YYYY-MM-DD` 形式を厳守（末尾の日付は週の最終日）
- **summary**: 50〜200文字の日本語要約（週の主要トレンドを含める）
- **tags**: 3個以上
- **provider**: 出力不要（後処理で自動注入される）
- **recap_period**: `"YYYY-MM-DD/YYYY-MM-DD"` 形式（開始日/終了日）
- **sources**: 記事中で引用した全 URL を漏れなく含める（https:// で始まる有効な URL のみ。**最低5件以上**）
- **automated**: 必ず `true`
- **sns_topics**: 記事内の主要トピックを3〜5個抽出。各トピックに `topic`（トピック名）と `summary`（100文字程度の要点）を記載。SNS投稿の素材として使用される
- ※ thumbnail フィールドは後処理で自動注入されるため、ここでは指定しない

### sources フィールドの厳守事項
- **sources は絶対に空にしないこと**。記事中で参照したすべての URL を含める
- 記事本文中の Markdown リンク `[text](url)` で使用した URL をすべて sources に含める
- 同じ URL の重複は不要

## 記事の文字量
- 本文（frontmatter を除く）は **5000文字以上** を目標とする
- 週のハイライトは十分に深掘りし、背景・影響・今後の見通しまで含める

## 書式ルール
- 段落は論理的なまとまりごとに構成し、1文ごとに改行しない
- 見出しの前後には空行を1行ずつ入れる
- **`$` 記号のエスケープ（厳守）**: コードブロック（```）やインラインコード（`` ` ``）の外で通貨記号 `$` を使う場合は、必ず `\$` とエスケープすること。例: `\$100`, `\$3/1Mトークン`。未エスケープの `$` は数式デリミタ（KaTeX）として解釈され、記事の表示が壊れる

## 記事構成

### 1. エグゼクティブサマリー
- 記事冒頭に3-5行で、今週の最重要トレンド・注目ポイントを要約する

### 2. 週のハイライト（最重要トピック3-5件）
各トピックについて以下の構造で記述（各500文字以上）:
- **概要**: 何が起きたか（入力記事の内容 + 続報）
- **背景と経緯**: なぜこれが重要か
- **技術的・社会的インパクト**: 業界・ユーザーへの影響
- **今後の展望**: 次週以降の注目ポイント
- **出典**: [情報源名](URL)

### 3. 週次トレンド分析
- 今週のAI業界全体を俯瞰したトレンド分析
- 複数ニュースに共通するテーマ・パターン
- 競合他社・プロジェクト間の動向比較（あれば）

### 4. 今後の展望
- 来週以降に注目すべき発表・イベント・動向
- 今週の出来事が中長期的に与える影響

### 5. 参考文献
以下の形式のテーブルを記事末尾に必ず記載する:

| タイトル | 情報源 | 日付 | URL |
|---------|--------|------|-----|
| ニュースタイトル1 | 発表元名 | YYYY-MM-DD | https://example.com/... |

## 品質基準（これを満たさない記事は不合格）
- 入力記事の活用: **全入力記事の内容を参照すること**
- ハイライト: **3件以上**（各500文字以上）
- sources 配列: **5件以上の有効な URL** を含めること
- 本文: **5000文字以上**
- recap_period: 正確な日付範囲を記載すること
