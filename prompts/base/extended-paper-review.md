## 調査対象領域（拡張10領域）

以下の10領域を調査対象とし、過去7日以内の論文を収集すること:

1. **ロボティクス・自律エージェント**（arXiv cs.RO）: ロボット工学、自律走行、ドローン、工業用ロボット等
2. **心理学・認知科学**: 認知心理学研究、人間の意思決定、神経科学の知見等
3. **経済学・行動経済学**: 経済政策、行動経済学研究、AI の経済的影響分析等
4. **生命科学・創薬AI**: タンパク質構造予測、分子生成モデル、創薬・医療AI等
5. **教育工学**: AI活用教育、学習支援システム、EdTech等
6. **経営学・組織論**: AI導入の組織変革、意思決定支援、ビジネス戦略等
7. **計算社会科学**: ソーシャルメディア分析、偽情報検出、社会シミュレーション等
8. **金融工学・計算ファイナンス**: アルゴリズム取引、リスク評価AI、FinTech等
9. **エネルギー工学・気候科学**: 電力需要予測、気候モデリング、再生可能エネルギー等
10. **宇宙工学・宇宙科学**: 衛星画像解析、宇宙探査AI、天文学的発見等

## 情報ソース

- **arXiv**: 各領域の対応カテゴリ（cs.RO, q-bio, econ, cs.CY 等）の最新投稿
- **学術ジャーナル・学会の公式サイト**: Nature, Science, PNAS, IEEE, ACM 等
- **各分野の研究機関・大学の公式発表・プレスリリース**

## 調査の進め方
1. 上記10領域それぞれについてarXivおよび学術ジャーナルを検索する
2. 過去7日以内に投稿・公開された論文の中から注目度・重要度が高いものを選定する
3. 各論文のアブストラクト・主要結果を確認する
4. 選定した論文群を1つの記事にまとめる

**日付制約（厳守）**: 過去7日以内に投稿・公開された論文のみを選定すること。1週間以上前の論文は選定しないこと。arXiv の投稿日（Submitted）または最終更新日を必ず確認すること。

**必須**: 検索は最低5回以上行い、領域ごとに検索キーワードを変えて横断的にリサーチすること。

## 出力形式
**出力は必ず以下の frontmatter 付き Markdown のみ。** 前置きテキスト・説明文・コードフェンスは一切付けないでください。
出力の最初の文字は `---` であること。

---
title: "拡張論文レビュー YYYY年MM月DD日 - [全体テーマを一言で]"
slug: "extended-paper-review-YYYY-MM-DD"
summary: "100-150文字の要約（50文字以上200文字以内）"
date: "YYYY-MM-DD"
tags: ["関連タグ1", "関連タグ2", "関連タグ3"]
category: "extended-paper-review"
automated: true
sources: ["論文URL1", "論文URL2", "論文URL3", "参考URL"]
---

### frontmatter ルール
- **slug**: `extended-paper-review-YYYY-MM-DD` 形式を厳守
- **summary**: 50〜200文字の日本語要約（取り上げた論文数と主要テーマを含める）
- **tags**: 3個以上
- **sources**: 記事中で引用した全 URL を漏れなく含める（https:// で始まる有効な URL のみ。**最低5件以上**）
- **automated**: 必ず `true`
- ※ thumbnail フィールドは後処理で自動注入されるため、ここでは指定しない

### sources フィールドの厳守事項
- **sources は絶対に空にしないこと**。記事中で参照したすべての URL を含める
- arXiv 論文は `https://arxiv.org/abs/XXXX.XXXXX` 形式の URL を使用すること
- 記事本文中の Markdown リンク `[text](url)` で使用した URL をすべて sources に含める
- 同じ URL の重複は不要

## 記事の文字量
- 本文（frontmatter を除く）は **7000文字以上** を目標とする
- 各論文の解説は **700文字以上**（論文紹介と平易な補足を合わせて）
- 読者が論文を読まなくても内容を理解できるようにする
- 具体的な数値（精度、パラメータ数、比較結果等）を積極的に含める

## 書式ルール
- 段落は論理的なまとまりごとに構成し、1文ごとに改行しない
- 論文解説と初学者向け補足は一つの流れとして自然に記述する
- 不確実な情報には断定を避ける表現を使う（「〜と考えられる」「〜の可能性がある」等）
- **`$` 記号のエスケープ（厳守）**: コードブロック（```）やインラインコード（`` ` ``）の外で通貨記号 `$` を使う場合は、必ず `\$` とエスケープすること。例: `\$100`, `\$3/1Mトークン`。未エスケープの `$` は数式デリミタ（KaTeX）として解釈され、記事の表示が壊れる

### 2. 注目論文（各領域から選定）

各論文について:
#### 論文 N: [論文タイトルの日本語訳]（[領域名]）

- **著者・所属**: 主要著者と所属機関

- **主要結果**: ベンチマーク名、スコア、比較対象、具体的数値（効果量、信頼区間を含む）

## 情報ソースの引用ルール
- 各論文への言及には必ず **Markdown リンク** で出典を明記すること
  - 例: [論文タイトル](https://arxiv.org/abs/XXXX.XXXXX)
- frontmatter の `sources` 配列にも全 URL を含めること
- **URL の正確性（厳守）**: 出典 URL は実在するページを指していること。存在しない URL や推測で構成した URL は絶対に使用しないこと。URL が正確か自信がない場合は、その URL を sources に含めないこと。

## 品質基準（これを満たさない記事は不合格）
- 論文数: **5件以上**（複数領域から選定）
- 各論文の解説: **700文字以上**
- sources 配列: **5件以上の有効な URL**（arXiv URL を優先）
- 本文: **7000文字以上**
- 情報源の多様性: 同一著者グループの論文だけでなく、異なる研究機関の論文を含めること
- 日付: 過去7日以内の論文のみ（1週間以上前の論文は不合格）
