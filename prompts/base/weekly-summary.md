## リサーチ指示

入力記事の各トピックについて、**web_search を使って最新の続報・進展を調査** してください。

### 調査の進め方
1. 入力記事を通読し、主要トピック（論文・ニュース・企業動向）を抽出する
2. 各トピックについて web_search で **続報・最新の進展** を検索する
   - 例: 論文レビューで取り上げた手法の後続研究・実用化の動き
   - 例: ニュースで報じた製品発表のその後の反応・競合の動き
3. 記事に含まれない **新たな重要トピック** が見つかった場合は追加で取り上げてよい
4. 検索は **最低5回以上** 行い、異なるキーワードで横断的にリサーチすること

### 調査対象ソース
- 企業公式発表・ブログ
- テックメディア（TechCrunch, The Verge, VentureBeat 等）
- AI専門メディア（AI News, Papers With Code 等）
- 日本語メディア（日経クロステック, ITmedia AI+ 等）
- arXiv（論文の続報・関連研究）

## 出力形式
**出力は必ず以下の frontmatter 付き Markdown のみ。** 前置きテキスト・説明文・コードフェンスは一切付けないでください。
出力の最初の文字は `---` であること。

---
title: "AI週次サマリ YYYY年MM月DD日週"
slug: "weekly-summary-YYYY-MM-DD"
summary: "100-150文字の要約（50文字以上200文字以内。取り上げたトピック数と主要テーマを含める）"
date: "YYYY-MM-DD"
tags: ["weekly-summary", "その他の集約タグ"]
category: "weekly-summary"
automated: true
provider: "claude"
summary_period: "YYYY-MM-DD/YYYY-MM-DD"
sources: ["外部URL1", "外部URL2", "外部URL3"]
---

### frontmatter ルール
- **slug**: `weekly-summary-YYYY-MM-DD` 形式を厳守（DD は公開日）
- **summary**: 50〜200文字の日本語要約
- **tags**: 第1要素は必ず `"weekly-summary"`。残りは入力記事のタグを集約（重複排除）
- **category**: 必ず `"weekly-summary"`
- **automated**: 必ず `true`
- **provider**: 必ず `"claude"`
- **summary_period**: `"開始日/終了日"`（集計対象期間。YYYY-MM-DD/YYYY-MM-DD 形式）
- **sources**: web_search で取得した **外部 URL のみ** を含める（自サイト記事URLは含めない）。**最低5件以上**
- ※ thumbnail フィールドは後処理で自動注入されるため、ここでは指定しない

### sources フィールドの厳守事項
- **sources は絶対に空にしないこと**。web_search で参照したすべての外部 URL を含める
- 自サイト（oct-rick-brick.com）の URL は sources に含めない（本文中のリンクで参照）
- 記事本文中の Markdown リンク `[text](url)` で使用した外部 URL をすべて sources に含める
- 同じ URL の重複は不要

## 記事の文字量
- 本文（frontmatter を除く）は **4000文字以上** を目標とする
- 各トピックについて十分な深掘りを行い、週全体の流れが把握できるようにする
- 入力記事の単なる要約ではなく、**横断的な分析と最新の続報** を加えた付加価値のあるレポートにすること

## 記事構成

### 1. エグゼクティブサマリー
- 記事冒頭に3-5行で、今週のAI業界のハイライト・注目すべきトレンドを要約する
- 入力記事の件数と主要テーマに言及する

### 2. 今週の注目トピック（入力記事をテーマ別にグルーピング）
関連する記事を3〜5つのテーマにグルーピングし、各テーマについて以下の構造で記述:

#### トピック N: [テーマ名]
- **今週の動き**: 関連する入力記事の内容を統合的に要約（300文字以上）
- **最新の進展**: web_search で判明した続報・新情報
- **分析**: このトピックの技術的・ビジネス的意義
- **参照記事**: 本サイトの関連記事へのリンク（タイトルと日付）
- **出典**: [外部情報源名](URL)

### 3. 来週の注目ポイント
- 今週のトレンドから予測される来週の注目事項
- 予定されているイベント・発表・リリース（web_search で確認）
- 継続的にフォローすべきテーマ

### 4. 参考文献
以下の形式のテーブルを記事末尾に必ず記載する:

| タイトル | 情報源 | 日付 | URL |
|---------|--------|------|-----|
| 外部記事タイトル1 | TechCrunch | YYYY-MM-DD | https://techcrunch.com/... |
| 外部記事タイトル2 | arXiv | YYYY-MM-DD | https://arxiv.org/abs/... |

**注意**: 参考文献テーブルには **外部 URL のみ** を含める。自サイト記事は本文中の「参照記事」で言及する。

## 情報ソースの引用ルール
- web_search で取得した外部情報には必ず **Markdown リンク** で出典を明記すること
  - 例: [TechCrunch の報道](https://techcrunch.com/...)
  - 例: [arXiv 論文](https://arxiv.org/abs/XXXX.XXXXX)
- 自サイトの入力記事を参照する場合は、タイトルと日付で言及する（URLリンクは不要）
  - 例: 「本サイト記事『AI論文ウィークリーレビュー 2026年3月15日』で取り上げた〜」
- frontmatter の `sources` 配列には **外部 URL のみ** を含めること

## 書式ルール
- **`$` 記号のエスケープ（厳守）**: コードブロック（```）やインラインコード（`` ` ``）の外で通貨記号 `$` を使う場合は、必ず `\$` とエスケープすること。例: `\$100`, `\$3/1Mトークン`。未エスケープの `$` は数式デリミタ（KaTeX）として解釈され、記事の表示が壊れる

## 品質基準（これを満たさない記事は不合格）
- トピック数: **3テーマ以上**
- 各トピック: **300文字以上**
- sources 配列: **5件以上の外部 URL**
- 本文: **4000文字以上**
- 情報源の多様性: **3つ以上の異なる外部ドメイン** からの情報を含めること
- web_search の実行: **5回以上**（入力記事だけで書かないこと）
- 自サイト記事の参照: 入力記事の **過半数以上** に言及すること
