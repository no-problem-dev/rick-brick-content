#### 論文1: Tool Verification for Test-Time Reinforcement Learning (T3RL)

- **著者・所属**: Ruotong Liao 他（所属機関は論文内で明記）
- **概要**:

テスト時強化学習（TTRL）は、ラベルなしのテスト入力に対して多数決による自己誘導報酬を通じてオンライン適応を可能にする、自己進化型大規模推論モデル（LRM）の新しいパラダイムとして登場した

。しかし従来手法には重大な欠陥がある。

誤った解答が頻繁に多数派を形成する「偽人気コンセンサス」が、検証されないまま報酬信号として強化され、誤ったモードへの収束を引き起こす

問題である。本論文はこの課題に対し、T3RL（Tool-Verification for Test-Time Reinforcement Learning）という新手法を提案している。

- **提案手法**:

T3RLは、報酬推定にテスト時ツール検証を導入する。具体的には、検証器が外部ツール（コード実行など）からの証拠を利用して、検証済みロールアウトを検証考慮型投票で上重み付けし、訓練用の信頼性の高い疑似ラベルを生成する

。このアプローチの核心は、

人間が内省だけでは信頼できない場合に外部証拠を求めるように、環境と相互作用してフィードバックを得ることで誤った仮説を除外する、外部検証メカニズムによって自己一貫性の閉じたループを破ることにある

。

  T3RLの処理フローは以下の通り：
  1. **生成器（Generator）**: 研究問題の候補解を提案
  2. **検証器（Verifier）**: ツール実行結果を用いて非形式的な自然言語メカニズムで欠陥や幻覚をチェック
  3. **修正器（Reviser）**: 検証器が特定したエラーを修正し、最終出力が承認されるまで反復

- **主要結果**:

様々な数学難易度（MATH-500、AMC、AIME 2024）および様々なバックボーンタイプにわたって、T3RLはTTRLを大幅に改善し、より難しい問題でより良いパフォーマンスを示した

。特に高難易度の数学オリンピック級問題において、従来のTTRLが頻繁に誤答の多数決に陥る一方、T3RLはコードインタープリタなどのツールによる検証を通じて正解率を顕著に向上させている。

  論文では、

DeepSeek-R1やOpenAIのo-seriesなど大規模推論モデルで強化学習が繰り返し進歩をもたらしてきたが、テスト時強化学習（TTRL）はラベル付きデータへの依存を取り除くことでさらに前進した。TTRLでは、LRMがまず複数の推論トレースを生成し、多数決により正解と仮定してラベルとし、RL報酬を導出する。LRMはラベルなしテスト入力に対する特定のオンラインRL訓練に基づいて進化する

と説明されている。

- **意義と限界**: T3RLの最大の意義は、外部検証という概念を導入することで、自己一貫性に基づく報酬の根本的脆弱性に対処したことにある。

自己一貫性ベースの報酬は根本的な脆弱性を露呈する。モデルの内部推論が偏っている場合、コンセンサスはもはや正確性と相関しない。さらに悪いことに、誘導された偏りは悪循環でさらに強化され増幅される。モデルは誤った推定にますます自信を深め、偽人気モード崩壊につながるまさにそのエラーを強化してしまう

。この問題は従来のTTRLの致命的欠陥であったが、T3RLはツール実行による客観的検証を組み込むことで、この悪循環を断ち切ることに成功している。

  限界としては、外部ツールの利用可能性に依存する点が挙げられる。数学問題ではコード実行による検証が有効だが、自然言語推論など検証が困難なドメインへの適用には課題が残る。また、ツール検証自体の信頼性も重要な要素となる。

- **出典**: [Tool Verification for Test-Time Reinforcement Learning](https://arxiv.org/abs/2603.02203)

#### 論文2: TTSR - Test-Time Self-Reflection for Continual Reasoning Improvement

- **著者・所属**: Haoyang He, Zihua Rong, Liangjie Zhao, Yunjia Zhao, Lan Yang, Honggang Zhang（所属機関は論文内で明記）
- **概要**:

テスト時訓練はテスト質問のみを使用したモデル適応を可能にし、大規模言語モデル（LLM）の推論能力を向上させる有望なパラダイムを提供する。しかし、テスト質問は非常に難しいことが多く、自己生成された疑似ラベルは信頼できず、既存の手法はモデルの特定の推論弱点に適応する効果的なメカニズムを欠いているため、非効率な学習につながる

という二つの主要課題に取り組んでいる。

  従来のTTRLやR-Zeroなどの手法は、単純な多数決による疑似ラベル生成に依存しているため、モデルが苦手とする特定のタイプの問題に対して効果的な学習が困難であった。TTSRはこの問題に対し、「自己反省」という人間的な学習プロセスをモデルに組み込むことで解決を図っている。

- **提案手法**:

TTSRは自己反省的なテスト時自己進化訓練フレームワークを提案する。TTSRは、テスト時に「生徒（Student）」と「教師（Teacher）」の役割を交互に切り替える単一の事前訓練言語モデルを採用する。生徒は問題解決と合成された変形問題からの学習に焦点を当て、一方で教師は生徒の失敗した推論軌跡を分析し、繰り返される推論の弱点を要約し、それに応じてターゲット化された変形問題を合成する

。

  TTSRの動作サイクルは以下の通り：
  1. **生徒フェーズ**: モデルがテスト問題に取り組み、複数の推論トレースを生成
  2. **教師フェーズ**: 失敗した推論トレースを分析し、共通の誤りパターンを特定
  3. **問題合成**: 特定された弱点に焦点を当てた類似問題を生成
  4. **継続学習**: 合成問題を用いてモデルを強化学習で訓練

  この反省駆動型合成アプローチにより、モデルは自身の弱点に特化した訓練データを動的に生成できる。

- **主要結果**:

数学ベンチマークにおいて、TTSRはテスト時強化学習ベースラインに対して大幅な向上を達成した。例えば、Qwen3-4B-Baseにおいて、TTSRはベースモデルと比較して平均10ポイント以上のパフォーマンス向上を達成し、特にAIME 2024、AIME 2025、Olympiadなどの高難易度タスクで大きな向上が見られた。注目すべきことに、AIMEベンチマークでの改善は10ポイントを超えており、反省誘導合成が深い多段階推論を必要とする問題に特に効果的であることを示している

。



一般推論においても、TTSRは一般推論ベンチマークでパフォーマンスを向上させた。すべてのモデルにわたって、TTSRはGPQA-DiamondとMMLU-Proで一貫した向上を達成し、改善は通常、最強ベースラインに対して3〜7ポイントの範囲であった

。この結果は、TTSRが数学に限らず幅広い推論タスクに適用可能であることを示している。

- **意義と限界**: TTSRの最大の革新は、単なる多数決による学習ではなく、メタ認知的な「反省」プロセスを導入したことにある。

このプロセスは、継続的な自己進化ループを通じて学習可能な領域内でモデルを改善するよう導く。複数の挑戦的な数学推論ベンチマークにおける実験結果は、TTSRが推論パフォーマンスを一貫して向上させ、異なるモデルバックボーンや一般領域推論タスクにわたってよく汎化することを示している。これらの発見は、教師が媒介する自己反省が、テスト時の安定的かつ継続的な推論改善のための効果的な経路を提供することを示唆している

。

  これは、AI研究において長らく課題とされてきた「モデルが自身の弱点を特定し、それを克服するための訓練データを自己生成する」という理想的な学習サイクルに、一歩近づいたと言える。

  限界としては、教師-生徒フレームワークの計算コストが挙げられる。反省と問題合成のプロセスは追加の推論を必要とするため、単純なテスト時推論よりも計算資源を消費する。また、生成される変形問題の品質がモデルの教師能力に依存するため、十分に成熟していないモデルでは効果が限定的である可能性がある。

- **出典**: [TTSR: Test-Time Self-Reflection for Continual Reasoning Improvement](https://arxiv.org/abs/2603.03297)

#### 論文3: Visual-ERM - Reward Modeling for Visual Equivalence

- **著者・所属**: Ziyu Liu, Shengyuan Ding, Xinyu Fang, Xuanlang Dai, Penghui Yang, Jianze Liang, Jiaqi Wang, Kai Chen, Dahua Lin, Yuhang Zang
- **概要**:

視覚からコードへのタスクは、チャート、表、SVGなどの構造化された視覚入力を、高い視覚的忠実度を持つ実行可能または構造化された表現に再構築することをモデルに要求する。最近の大規模視覚言語モデル（LVLM）は教師あり微調整を通じて強力な結果を達成しているが、報酬信号のミスアライメントのため強化学習は困難なままである。既存の報酬は、テキストルールまたは粗い視覚埋め込み類似性に依存しており、いずれも細粒度の視覚的不一致を捉えることができず、報酬ハッキングに対して脆弱である

。

  視覚からコードへの変換タスク（チャートやテーブルの構造をコードで再現するなど）において、従来の報酬モデルは大きく二つのカテゴリに分かれる：
  1. **テキストベース報酬**: 編集距離やTEDSなどの指標を使用するが、レイアウト、配置、間隔、スタイルなどの重要な視覚的手がかりを無視する
  2. **視覚埋め込み報酬**: DINO類似性などを使用するが、粗粒度で意味的に偏っており、報酬ハッキングに脆弱である

- **提案手法**:

Visual Equivalence Reward Model（Visual-ERM）を提案する。これは、レンダリングされた視覚空間で直接視覚からコードへの品質を評価するための、細粒度で解釈可能でタスク非依存のフィードバックを提供するマルチモーダル生成型報酬モデルである

。Visual-ERMの重要な特徴は、報酬をスカラー値として出力するのではなく、視覚的不一致を詳細に記述する生成型アプローチを採用している点にある。



グローバル構造とローカル視覚詳細を共同でモデリングすることにより、Visual-ERMは三つの重要な特性を持つ報酬信号を提供する：（i）細粒度：粗い意味的類似性を超えた微妙な視覚的不一致を捉える、（ii）解釈可能：テスト時スケーリングのための反省と修正を導く診断的フィードバックを生成する、（iii）タスク非依存：単一のRMがチャートからコード、テーブルからMarkdown、SVGからコードにわたって汎化する。これにより、テキストベースのメトリクスや視覚エンコーダー類似性よりも実質的に強力な監視が可能となり、真の視覚的等価性への忠実なガイダンスを提供する

。

- **主要結果**:

強化学習に統合されたVisual-ERMは、チャートからコードでQwen3-VL-8B-Instructを+8.4改善し、テーブルとSVG解析で一貫した向上をもたらした（平均+2.7、+4.1）、さらに反省を通じてテスト時スケーリングを強化する

。

  Visual-ERMの評価のため、

VisualCritic-RewardBench（VC-RewardBench）という、チャート、テーブル、SVGにわたる細粒度の画像間不一致判定を評価するベンチマークを導入した。VC-RewardBenchにおいて、Visual-ERMは8Bスケールにもかかわらず強力なパフォーマンスを達成し、Qwen3-VL-235B-Instructを決定的に上回り、主要なクローズドソースモデルに接近している

。これは、適切に設計された報酬モデルが、モデルサイズの制約を超えて高い性能を達成できることを示している。

  さらに重要なのは、

Visual-ERMの価値はRLの報酬モデルとして機能することを超えている。モデル反省のための細粒度で解釈可能なフィードバックを提供し、テスト時スケーリング（TTS）が再構築出力をさらに洗練することを可能にする。そのため、Visual-ERMを推論ループに組み込んだ：Visual-ERMが各候補を批評し、ポリシーモデルがそれに応じて出力を更新する。Visual-ERMをテスト時スケーリングパイプラインに統合すると、ベースのQwen3-VL-8B-Instructで+8.0、3回の反省ラウンドでRLチューニングされたモデルで+3.1の追加利得がもたらされる

ことが示された。

- **意義と限界**: Visual-ERMの最大の意義は、視覚タスクにおける報酬モデリングのパラダイムを転換したことにある。従来のスカラー報酬ではなく、詳細な視覚的不一致記述を生成することで、モデルは単に「良い」「悪い」だけでなく、「どこが」「どのように」間違っているかを理解できるようになった。これは、人間がコードレビューで行うような詳細なフィードバックプロセスをAIが模倣できることを意味する。

  報酬ハッキングへの耐性も重要な利点である。従来の視覚埋め込みベースの報酬では、モデルが表面的な視覚的類似性を最大化する一方で、実際の構造的正確性を犠牲にする「報酬ハッキング」が頻繁に発生していた。Visual-ERMは、レンダリング後の実際の視覚出力を比較することで、この問題を緩和している。

  限界としては、生成型アプローチの計算コストが挙げられる。スカラー報酬を計算するよりも、詳細な記述テキストを生成する方が時間がかかる。また、評価の客観性についても課題がある。論文では、

LLMを使用すると、ジャッジの較正と推論スタイルへの潜在的依存性が生じる。しかし、VC-RewardBenchではジャッジは比較的制約された照合を実行するよう求められているだけであり（オープンエンドの好み順位付けではなく）、特定のジャッジモデルに対する感度が低いと仮定される

と説明されているが、完全な客観性の保証は困難である。

- **出典**: [Visual-ERM: Reward Modeling for Visual Equivalence](https://arxiv.org/abs/2603.13224)

#### 論文4: TaoBench - 自動定理証明LLMの汎化性能評価

- **著者・所属**: Alexander K Taylor, Junyi Zhang, Ethan Ji, Vigyan Sahai, Haikang Deng, Yuanzhou Chen, Yifan Yuan, Di Wu, Jia-Chen Gu, Kai-Wei Chang, Nanyun Peng, Amit Sahai, Wei Wang
- **概要**: 自動定理証明（ATP）は、数学的推論能力を評価する最も厳密なベンチマークの一つとして注目されている。

23rd European Semantic Web Conference（ESWC）、ESWC 2026で受理された。TaoBenchは自動定理証明LLMがMathLib以外に汎化するかをテストする

。従来の評価では、主にLean証明アシスタントのMathLibライブラリに含まれる定理に焦点が当てられていたが、これらは訓練データに含まれている可能性があり、真の汎化性能を測定できていない懸念があった。

- **提案手法**: TaoBenchは、MathLib以外の数学領域から収集された定理を用いて、自動定理証明LLMの汎化性能を評価する新しいベンチマークである。これにより、モデルが訓練時に見たことのない数学的構造に対してどの程度の推論能力を発揮できるかを測定できる。

  ベンチマークの設計において重要なのは、以下の点である：
  1. **訓練データとの分離**: MathLibに含まれない定理を厳選
  2. **多様性**: 異なる数学分野からの定理を含む
  3. **難易度の段階性**: 基礎的な定理から高度な定理まで幅広くカバー

- **主要結果**: 論文の詳細な結果は公開情報から完全には確認できないが、従来のMathLib中心の評価では高性能を示していたモデルが、TaoBenchでは大幅に性能が低下することが示唆されている。これは、現在のATPモデルが真の数学的推論能力というよりは、訓練データのパターンマッチングに依存している可能性を示唆している。

- **意義と限界**: TaoBenchの意義は、自動定理証明AIの真の能力を測定する信頼性の高い評価基準を提供したことにある。従来の評価では、モデルが訓練データに含まれる定理を「記憶」している可能性を排除できなかったが、TaoBenchはこの問題に対処している。

  これは、Google DeepMindのAletheiaなど、

研究論文（Feng26）が算術幾何学に関して完全に人間の介入なしで生成されたことを含む、いくつかの「初」を達成したシステムの成果を、より厳密に評価するための基盤を提供する。また、Erdős予想データベースから4つのオープン問題を自律的に解決した

といった主張の妥当性を検証する上でも重要である。

  限界としては、ベンチマーク自体の構築における主観性が挙げられる。「MathLib以外」という基準は明確だが、どの定理を選ぶかによって難易度や評価結果が変わる可能性がある。また、数学の新しい分野が次々と開拓される中で、ベンチマークの継続的更新が必要となる。
